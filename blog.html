• Discuss the scope of var, let, and const

    • => Variable declaration using var is globally scoped which means it can be accessed from anywhere inside the whole .js file. Meanwhile, let and const are block scoped. So, if we want to access these from an outer block, then it will raise an error.


  • Tell us the use cases of null and undefined

    • => 'undefined' refers such case when a variable is declared but no value is assigned to it. Moreover, if no. of parameters is les than no. of arguments then remaining parameters are by default initialized as undefined and it's also true for by default return value in case of function. After then, value of a object property which doesn't exist is also undefined. On the other hand, null is used to set value in a variable explicitly or intentionally. It helps to compare and check with other values. Lastly, both null and undefined are used to check falsy statement.


  • What do you mean by REST API?

    • => REST stands for Representational State Transfer whereas API means Application Programming Interface. Generally, it use the standard HTTP methods for CRUD (Create, Read, Update, Delete) operations sent by client-

      GET: Retrieve data from the server.
      POST: Create a new resource on the server.
      PUT/PATCH: Update an existing resource or create one if it doesn't exist.
      DELETE: Remove a resource from the server.
